00 Why This Paper Changed Everything

In June 2017, eight Google researchers published a paper that would reshape artificial intelligence. Before this paper, the best language models used recurrent neural networks (RNNs) — processing words one at a time, like reading a book letter by letter.

The Transformer introduced a radical idea: what if a model could look at all words simultaneously and decide which ones matter for each other?

💡 The Key Insight

You don't need recurrence (processing one word at a time). You don't need convolution. Attention alone — letting every word "look at" every other word — is sufficient to understand language.

Every major AI model in 2026 — GPT-4, Claude, Gemini, LLaMA — is built on this architecture. Understanding this paper is understanding the foundation of modern AI.

Let's build up to it, from the absolute basics.

01 Math Prerequisites

Before diving into the Transformer, you need to be comfortable with a few mathematical tools. Don't worry — we'll build every concept from intuition, not memorization.

1.1 — Vectors: The Language of AI

🎯 Analogy

A vector is a list of numbers that describes something. Think of it like a GPS coordinate — but instead of 2 numbers (latitude, longitude), we use hundreds of numbers to describe the "location" of a word in meaning-space.

A vector is an ordered list of numbers. In AI, we use vectors to represent everything — words, sentences, images, concepts.

v⃗ = [0.2, -0.5, 0.8, 0.1, -0.3]   ← a 5-dimensional vector
Each number captures one "feature" of whatever we're representing. For a word like "king," one dimension might encode "royalty," another "gender," another "power." The model learns what each dimension means.

Why vectors matter for Transformers: Every word in a sentence is converted into a vector (called an embedding). The Transformer then manipulates these vectors to understand meaning.

1.2 — Matrices: Transforming Vectors

🎯 Analogy

If a vector is a point in space, a matrix is a machine that takes points and moves them to new locations. Multiplying a vector by a matrix transforms it — rotates, stretches, or projects it into a different space.

A matrix is a grid of numbers. When you multiply a matrix W by a vector x, you get a new vector y.

W · x = y

┌ 0.1 0.3 ┐ ┌ 2.0 ┐ ┌ 0.1×2 + 0.3×1 ┐ ┌ 0.5 ┐ │ 0.5 0.2 │ · │ 1.0 │ = │ 0.5×2 + 0.2×1 │ = │ 1.2 │ └ 0.4 0.6 ┘ └ ┘ └ 0.4×2 + 0.6×1 ┘ └ 1.4 ┘
Each row of W performs a dot product with x to produce one number in y. This is how neural networks transform data — every layer multiplies inputs by a learned weight matrix W.
🧠 Intuition — Why This Matters

In the Transformer, there are three critical matrices: WQ, WK, WV. They transform each word's embedding into a Query, a Key, and a Value. This single operation is the heart of attention.

1.3 — Dot Product: Measuring Similarity

🎯 Analogy

The dot product tells you "how much do these two vectors point in the same direction?" If two word vectors have a high dot product, the words are related. If it's zero, they're unrelated. If negative, they're opposites.

a⃗ · b⃗ = a₁b₁ + a₂b₂ + ... + aₙbₙ

[1, 0, 2] · [3, 1, 1] = 1×3 + 0×1 + 2×1 = 5
[1, 0, 0] · [0, 1, 0] = 1×0 + 0×1 + 0×0 = 0 ← perpendicular = unrelated
In attention: The dot product between a Query vector (what am I looking for?) and a Key vector (what do I contain?) produces an attention score — how relevant is word B to word A?

1.4 — Softmax: Turning Scores into Probabilities

🎯 Analogy

Imagine you rated 4 restaurants: scores [3, 1, 0.5, 5]. Softmax converts these raw scores into probabilities that sum to 1 — like saying "I'm 64% likely to pick restaurant #4, 9% for #1, etc." It amplifies the differences: winners win bigger, losers shrink more.

softmax(zᵢ) = ezᵢ / Σⱼ ezⱼ

Input: [3.0, 1.0, 0.5, 5.0]
e^z: [20.1, 2.7, 1.6, 148.4] ← exponentiate each
Sum: 172.8
Output: [0.12, 0.02, 0.01, 0.86] ← probabilities sum to 1
In attention: After computing dot products between Q and K, softmax converts them into attention weights — a probability distribution over all words, telling the model "pay 86% attention to word 4, 12% to word 1, nearly ignore the rest."

1.5 — Layer Normalization: Keeping Numbers Stable

As data flows through many layers, the numbers can grow very large or very small (exploding/vanishing gradients). Layer normalization rescales values to have mean ≈ 0 and standard deviation ≈ 1 at each layer.

LayerNorm(x) = γ · (x - μ) / (σ + ε) + β
μ = mean of x, σ = std deviation, ε = tiny number to prevent division by zero.
γ (scale) and β (shift) are learned — the model can undo normalization if needed.

1.6 — Prerequisite Summary

📐 Vectors

Lists of numbers that represent words/concepts in high-dimensional space

📊 Matrices

Grids of numbers that transform vectors. Neural network weights are matrices

🎯 Dot Product

Measures similarity between two vectors. Core of attention scoring

📈 Softmax

Converts raw scores to probabilities. Makes attention weights sum to 1

⚖️ Layer Norm

Normalizes values to prevent numbers from exploding/vanishing across layers

📉 Gradient Descent

Algorithm to update weights by moving in the direction that reduces error

02 Deep Learning Foundations

Now that you have the math tools, let's understand the building blocks of neural networks that the Transformer is built from.

2.1 — What Is a Neuron?

🎯 Analogy

A neuron is a tiny decision-maker. It receives several inputs, multiplies each by a weight (how important is this input?), adds a bias (default preference), then passes the result through an activation function (should I fire or not?).

y = activation(w₁x₁ + w₂x₂ + ... + wₙxₙ + b)

= activation(W · x + b)
W = weights (learned), b = bias (learned), x = input. The activation function introduces non-linearity — without it, stacking layers would be pointless (a stack of linear operations = one linear operation).

Common Activation Functions

ReLU(x) = max(0, x) ← simple, fast, most common
GELU(x) = x · Φ(x) ← smooth ReLU, used in Transformers
Sigmoid(x) = 1 / (1 + e⁻ˣ) ← squashes to [0, 1]
Tanh(x) = (eˣ - e⁻ˣ)/(eˣ + e⁻ˣ) ← squashes to [-1, 1]
The Transformer uses GELU in its feed-forward layers — it's smoother than ReLU and handles negative values better. The original paper uses ReLU, but GELU became standard.

2.2 — Embeddings: Words as Vectors

Computers can't read words. We need to convert each word into a vector of numbers — this is called an embedding.

🎯 Analogy

Imagine a city map where every word has a location. "King" is near "Queen" and "Ruler." "Dog" is near "Cat" and "Pet." "King" is far from "Toaster." Embeddings are these coordinates — words with similar meanings have similar vectors.

ANIMATED — Words clustering by meaning in vector space
Embedding("king") → [0.2, -0.5, 0.8, 0.1, ...] ← 512 numbers
Embedding("queen") → [0.3, -0.4, 0.7, 0.3, ...] ← nearby in vector space!
Embedding("car") → [0.9, 0.1, -0.6, 0.7, ...] ← far away

Famous result: king - man + woman ≈ queen
In the Transformer, the embedding layer is a learned lookup table: each word in the vocabulary (e.g., 50,000 words) maps to a 512-dimensional vector. These vectors are trained — the model learns the best "addresses" for each word.

2.3 — Backpropagation: How Networks Learn

A neural network learns by:

1
Forward Pass

Feed input through all layers → get prediction

2
Compute Loss

Compare prediction to correct answer → how wrong are we?

3
Backward Pass

Use chain rule of calculus to compute: "how much did each weight contribute to the error?"

4
Update Weights

Nudge each weight in the direction that reduces the error: w = w - lr × ∂Loss/∂w

🧠 Intuition

Backpropagation is asking: "For each knob (weight) in the machine, which way should I turn it to make the output less wrong?" The chain rule lets us compute this efficiently for millions of weights simultaneously.

2.4 — Cross-Entropy Loss: The Transformer's Error Signal

For language models, the loss function is cross-entropy — it measures how surprised the model is by the correct answer.

Loss = -log(P(correct word))

If model predicts P("the") = 0.9, correct = "the" → Loss = -log(0.9) = 0.105 ← low, good!
If model predicts P("the") = 0.01, correct = "the" → Loss = -log(0.01) = 4.605 ← high, bad!
When the model is confident and correct, loss is near 0. When it's wrong, loss is huge. The model learns to minimize this across all training examples.

03 The Sequence-to-Sequence Problem

The Transformer was designed for sequence-to-sequence tasks — turning one sequence into another. Translation is the classic example:

Input: "The cat sat on the mat" (English) ↓ [ MODEL ] ↓ Output: "Le chat était assis sur le tapis" (French)
Seq2seq: variable-length input → variable-length output

3.1 — Why RNNs Failed

Before Transformers, we used Recurrent Neural Networks (RNNs/LSTMs). They processed words sequentially — one at a time, left to right:

"The" → [RNN] → h₁ → "cat" → [RNN] → h₂ → "sat" → [RNN] → h₃ → ... ↓ The final hidden state h₃ must compress the ENTIRE sentence into one vector!
RNNs create a "bottleneck" — all meaning compressed into a single vector

Three fatal problems:

🐌 Sequential = Slow

Must process word 1 before word 2. Can't parallelize across GPUs. Training is painfully slow.

🧠 Forgetting

By the time you reach word 100, you've largely forgotten word 1. Long-range dependencies vanish.

📦 Bottleneck

The entire input meaning must be compressed into one fixed-size vector. Information is inevitably lost.

💡 The Transformer Solution

What if instead of reading sequentially, the model could look at all words at once and let each word decide which other words are relevant to it? This is self-attention — and it's fully parallelizable.

04 Self-Attention — The Core Idea

This is the most important section. If you understand this, you understand Transformers.

4.1 — The Library Analogy

🎯 Analogy — Attention as a Library

Imagine you're in a library. You have a question (Query). Each book on the shelf has a title (Key) and content (Value). You compare your question to every title, find the most relevant books, and read a weighted mix of their content. That's attention.

For each word in a sentence, self-attention computes three vectors:

Q — Query

"What am I looking for?" — what this word wants to find in other words

K — Key

"What do I contain?" — what this word advertises about itself

V — Value

"Here's my actual content" — the information to pass along if selected

4.2 — Computing Q, K, V

Each word's embedding x is multiplied by three learned weight matrices:

Q = x · WQ     K = x · WK     V = x · WV
If x is 512-dimensional and we project to d_k = 64, then each W matrix is 512 × 64. The model learns these matrices during training — learning what makes a good query, key, and value for each word.

4.3 — The Attention Formula

Attention(Q, K, V) = softmax( Q · KT / √dk ) · V

Let's break this down step by step:

1
Q · KT — Compute Similarity

Dot product of every Query with every Key. Result: a matrix of "how much should word i attend to word j?"

2
÷ √dk — Scale Down

Divide by √64 = 8. Without this, dot products grow too large → softmax becomes too "spiky" (one word gets 99.9% attention). This keeps gradients flowing.

3
softmax() — Normalize to Probabilities

Convert raw scores to probabilities (sum to 1). Now each word has a probability distribution over all other words.

4
× V — Weighted Sum of Values

Multiply attention weights by Values. Each word's output = weighted mix of all other words' Values, weighted by relevance.

4.4 — Worked Example

Sentence: "The cat sat on the mat"

Let's trace what happens for the word "sat":

Word: "sat" Step 1: Create Query for "sat" → Q_sat = x_sat · W^Q = "What did the action?" Step 2: Create Keys for ALL words: K_the = x_the · W^K K_cat = x_cat · W^K K_sat = x_sat · W^K K_on = x_on · W^K K_the2 = x_the · W^K K_mat = x_mat · W^K Step 3: Dot product Q_sat with each Key: Q_sat · K_the = 1.2 (low) Q_sat · K_cat = 8.5 ← HIGH! "cat" is relevant to "sat" Q_sat · K_sat = 3.1 (moderate - self) Q_sat · K_on = 2.0 (low) Q_sat · K_the2 = 0.8 (very low) Q_sat · K_mat = 6.2 ← HIGH! "mat" is relevant to "sat" Step 4: Scale by √64 = 8, then softmax: Scores: [0.15, 0.15, 1.06, 0.56, 0.25, 0.39, 0.10, 0.46] After softmax: [0.04, 0.42, 0.10, 0.06, 0.03, 0.35] the cat sat on the mat Step 5: Output for "sat" = 0.04·V_the + 0.42·V_cat + 0.10·V_sat + 0.06·V_on + 0.03·V_the + 0.35·V_mat
Result: "sat" now carries information mostly about "cat" (42%) and "mat" (35%) — it learned WHO sat and WHERE!

05 Multi-Head Attention

🎯 Analogy

One attention head asks one type of question. But language has many relationships: syntax (who did what), semantics (what does it mean), coreference (who does "it" refer to). Multi-head attention is like having 8 different experts, each noticing different patterns, then combining their insights.

Instead of one large attention operation, the Transformer runs h = 8 parallel attention heads, each with its own WQ, WK, WV:

headi = Attention(x·WiQ, x·WiK, x·WiV)

MultiHead = Concat(head₁, head₂, ..., head₈) · WO
If d_model = 512 and h = 8 heads, each head works in d_k = 512/8 = 64 dimensions. After concatenation (8 × 64 = 512), we multiply by WO (512×512) to mix the heads' outputs. Total computation cost ≈ same as single large attention.
Input (512-dim) │ ┌────┼────┬────┬────┬────┬────┬────┬────┐ │ │ │ │ │ │ │ │ │ ▼ ▼ ▼ ▼ ▼ ▼ ▼ ▼ ▼ Head Head Head Head Head Head Head Head Head 1 2 3 4 5 6 7 8 (64d)(64d)(64d)(64d)(64d)(64d)(64d)(64d) │ │ │ │ │ │ │ │ └────┴────┴────┴────┴────┴────┴────┘ │ Concatenate → 512-dim │ × W^O │ Output (512-dim)
Each head learns a different "type of relationship" — one might learn syntax, another semantics, another position

06 Positional Encoding

Problem: Unlike RNNs, attention treats all words as a set — it has no concept of order. "The cat sat on the mat" and "mat the on sat cat the" look identical to vanilla attention.

Solution: Add a unique "position signal" to each word's embedding before feeding it to the Transformer.

PE(pos, 2i) = sin(pos / 100002i/d_model)
PE(pos, 2i+1) = cos(pos / 100002i/d_model)
pos = position in the sentence (0, 1, 2, ...)
i = dimension index (0 to 255 for d_model=512)
Even dimensions use sin, odd dimensions use cos. Different frequencies at different dimensions let the model learn to attend to relative positions.
🧠 Why Sinusoids?

For any fixed offset k, PE(pos+k) can be expressed as a linear function of PE(pos). This means the model can easily learn "3 words ago" or "5 words ahead" — relative positions are encoded naturally. Think of it like a clock: the hour hand and minute hand together tell you both absolute time and elapsed time.

07 The Full Transformer Architecture

Now we assemble all the pieces. The Transformer has two halves: an Encoder (reads the input) and a Decoder (generates the output).

7.1 — Encoder Block (×6 layers)

Each encoder layer has two sub-layers:

1
Multi-Head Self-Attention

Every input word attends to every other input word. "sat" discovers it relates to "cat" and "mat."

2
Feed-Forward Network (FFN)

Two linear layers with GELU activation: FFN(x) = W₂ · GELU(W₁ · x + b₁) + b₂. The inner dimension is 4× the model dimension (512 → 2048 → 512). This is where the model stores "knowledge."

Both sub-layers use residual connections and layer normalization:

output = LayerNorm(x + SubLayer(x))
The residual connection (x + result) lets gradients flow directly through the network. Without it, training 6+ layers would be nearly impossible. Think of it as a "highway" for information.

7.2 — Decoder Block (×6 layers)

Each decoder layer has three sub-layers:

1
Masked Self-Attention

Same as encoder, but with a mask: each output word can only attend to previous output words. The model can't "cheat" by looking at future words it hasn't generated yet.

2
Cross-Attention

Queries come from the decoder, but Keys and Values come from the encoder output. This is how the decoder reads the input: "Based on what I've generated so far (Q), which parts of the input (K, V) should I focus on?"

3
Feed-Forward Network

Same as encoder — 2-layer FFN with residual + layer norm.

💡 Why the mask?

During training, we feed the entire correct output at once (for speed). The mask prevents position i from seeing positions > i, simulating autoregressive generation. Without it, the model would just memorize the answer instead of learning to predict it.

08 Training the Transformer

8.1 — The Training Objective

For each training example, the model predicts the next word at every position, and we minimize cross-entropy loss across all positions:

Loss = - Σt log P(wordt | word1, ..., wordt-1)
Sum of negative log-probabilities at every position. Lower = better predictions.

8.2 — Key Training Details from the Paper

📐 d_model = 512

Embedding dimension — every vector in the model is 512-dimensional

🔢 h = 8 heads

8 parallel attention heads, each with d_k = d_v = 64

📚 N = 6 layers

6 encoder layers + 6 decoder layers stacked

📈 d_ff = 2048

Inner dimension of FFN = 4× model dimension

⚡ Adam Optimizer

With warmup: LR increases for 4000 steps, then decays proportional to 1/√step

🎲 Dropout = 0.1

Applied to attention weights, FFN, and embeddings to prevent overfitting

8.3 — Label Smoothing

Instead of training with "100% confident" targets, the paper uses label smoothing (ε=0.1): the target is 90% on the correct word and 10% spread across all other words. This makes the model less overconfident and improves generalization.

09 Why It Still Matters in 2026

The Transformer from 2017 is the direct ancestor of every frontier model:

GPT-4 / GPT-4o

Decoder-only Transformer with trillions of parameters

Claude 3.5

Constitutional AI built on Transformer backbone

Gemini 2.0

Multimodal Transformer with MoE (Mixture of Experts)

LLaMA 3

Open-source Transformer with RoPE + GQA

The core ideas — Q/K/V attention, multi-head parallelism, positional encoding, residual connections — are unchanged. What has evolved:

RoPE replaced sinusoidal PE

Rotary position embeddings handle longer contexts better

GQA replaced MHA

Grouped-Query Attention reduces KV cache memory

Pre-norm replaced post-norm

LayerNorm before (not after) attention stabilizes training

SwiGLU replaced ReLU

Better activation function in FFN layers

🧠 Key takeaway

If you deeply understand this 2017 paper, you understand the skeleton of every 2026 model. The innovations since then are optimizations — the architecture is fundamentally the same. This paper is your foundation. Build on it.
