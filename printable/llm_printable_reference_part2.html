Linear Algebra — Must Know
Matrix multiply
(AB)ᵢⱼ = Σₖ AᵢₖBₖⱼ
dims: (m×n)(n×p)→(m×p)
Dot product
a·b = Σᵢ aᵢbᵢ = ‖a‖‖b‖cos(θ)
Eigenvectors
Av = λv
v = eigenvec, λ = eigenval
Transpose rules
(AB)ᵀ = BᵀAᵀ
(A+B)ᵀ = Aᵀ+Bᵀ
Calculus — Chain Rule + Gradients
Chain rule (fundamental for backprop)
dz/dx = (dz/dy)(dy/dx)
∂L/∂x = (∂L/∂y)(∂y/∂x)
Gradient of common functions
∂(Wx)/∂W = xᵀ ∂(Wx)/∂x = Wᵀ
∂(‖x‖²)/∂x = 2x ∂(aᵀx)/∂x = a
Softmax gradient (derive in interviews)
∂σᵢ/∂zⱼ = σᵢ(δᵢⱼ - σⱼ)
δᵢⱼ = 1 if i=j, else 0
When i=j: σᵢ(1-σᵢ). When i≠j: -σᵢσⱼ. The Jacobian of softmax is a full matrix, not a diagonal.
Probability — LLM-Relevant Concepts
Bayes theorem
P(A|B) = P(B|A)P(A)/P(B)
KL divergence
KL(P‖Q) = Σ P(x)log(P(x)/Q(x))
≥ 0, not symmetric
Cross-entropy loss
H(y,ŷ) = -Σ yᵢlog(ŷᵢ)
= NLL for LM training
MLE objective
θ* = argmax Σ log P(xᵢ|θ)
= argmin NLL
Backprop on 2-layer MLP — Step by Step
FForward: z₁=W₁x+b₁, a₁=ReLU(z₁), z₂=W₂a₁+b₂, ŷ=softmax(z₂)
LLoss: L = -Σ yᵢlog(ŷᵢ) (cross-entropy)
1∂L/∂z₂: = ŷ - y (softmax + CE gradient, clean result)
2∂L/∂W₂: = (∂L/∂z₂) · a₁ᵀ    ∂L/∂b₂: = ∂L/∂z₂
3∂L/∂a₁: = W₂ᵀ · (∂L/∂z₂)
4∂L/∂z₁: = (∂L/∂a₁) ⊙ ReLU'(z₁)   [⊙ = elementwise, ReLU'=1 if z>0]
5∂L/∂W₁: = (∂L/∂z₁) · xᵀ    ∂L/∂b₁: = ∂L/∂z₁
Numerical gradient check: ∂L/∂w ≈ [L(w+ε) - L(w-ε)] / 2ε. Use ε=1e-5. Analytical and numerical gradients should match to 1e-5 precision.

How to Read and Annotate ML Papers
Three-pass method · What to annotate · How to take notes
All months · Paper reading
The Three-Pass Method
1Pass 1 — 10 minutes: Read title, abstract, intro, conclusion, all figure captions. Decide: is this worth a full read? What problem does it solve?
2Pass 2 — 1 hour: Read fully but skip dense proofs. Understand every figure. Mark equations you don't understand. Write the core idea in one sentence in the margin.
3Pass 3 — 2–4 hours (print-worthy papers only): Derive every equation. Understand every proof. Implement from scratch. Find at least one flaw or open question.
For this roadmap: do Pass 3 only on the 7 Tier-1 print papers. Do Pass 2 on the remaining 30–40 papers. Do Pass 1 on papers you're skimming for breadth.
What to Write in Margins
  • ✓ — I understand this. Can re-derive.
  • ? — I don't fully understand. Come back.
  • ! — Important result. Likely interview question.
  • ★ — Key insight. Worth memorizing.
  • →impl — I should implement this.
  • cf. — Compare with [other paper].
  • Circle undefined symbols. Write definitions in margin.
  • Draw alternative diagrams if figures are unclear.
  • Write the "why" next to every equation — not just what.
After Reading — Feynman Test
Close the paper. Explain the core idea out loud as if teaching a first-year student. If you stumble, go back. You only truly understand what you can teach.

ProblemWhat problem does this paper solve? What was broken before?
Key ideaOne sentence. What is the novel contribution?
MethodHow does it work? Key equations + intuition.
ResultsWhat did they show? Key table/figure numbers.
LimitationsWhat doesn't this solve? What follow-up papers address this?

BPE Tokenizer — How It Works
1Initialize: character vocabulary. Each word = sequence of chars + end-of-word token ⟨/w⟩
2Count pairs: find most frequent adjacent pair in corpus (e.g. "t h" appearing 800 times)
3Merge: create new token "th". Replace all "t h" with "th" in corpus.
4Repeat for N merge operations (GPT-2: 50,000 merges)
BPE balances vocabulary size vs token length. Common words get single tokens. Rare words split into subwords. Unknown words handled via character fallback.

BERT Pre-Training Objectives
Masked Language Modeling (MLM)
15% of tokens replaced:
80% → [MASK], 10% → random, 10% → unchanged
Predict original token from context
MLM forces bidirectional context — each token can attend to all others. Unlike GPT (left-to-right only).
Next Sentence Prediction (NSP)
Input: [CLS] Sentence A [SEP] Sentence B [SEP]
Predict: IsNext (50%) or NotNext (50%)
NSP is now considered less useful — RoBERTa removed it and improved BERT. But know it for the BERT paper.

Key Interview Questions
Why does BERT use bidirectional attention while GPT uses causal?
BERT is designed for understanding tasks (classification, QA) where full context is available. GPT is designed for generation where future tokens aren't available at decode time. Different objectives → different attention masks.
Why replace 10% of masked tokens with random tokens in MLM?
Without random replacement, the model learns that [MASK] tokens always need prediction and ignores non-masked tokens during fine-tuning. Random tokens force the model to maintain good representations of all tokens.

W9 · Data
Data Quality Hierarchy (LIMA + Phi-1 insight)
1000 expert-curated examples > 100,000 scraped examples. Quality signals: diversity, correctness, clarity, appropriate difficulty. Quantity is the last lever to pull.
ChatML Format — Standard Fine-Tuning Structure
<|im_start|>system
You are a helpful assistant.<|im_end|>
<|im_start|>user
What is LoRA?<|im_end|>
<|im_start|>assistant
LoRA is...<|im_end|>
Used by Mistral, LLaMA-3, Qwen. Each conversation has system/user/assistant roles. im_start/im_end are special tokens added to vocabulary.
Self-Instruct Pipeline (Wang et al., 2022)
1Seed tasks: 175 manually written instruction-response pairs
2Task generation: GPT samples 8 random seeds → generates 20 new instructions
3Filter: remove exact duplicates, low-quality, non-English
4Response gen: GPT generates response for each instruction
5Fine-tune: use instruction-response pairs to fine-tune base model
Alpaca (Stanford) applied Self-Instruct to LLaMA using GPT-3.5 — generated 52K examples for $500. Result was competitive with InstructGPT-175B at the time.
MinHash Near-Deduplication
1Shingling: convert each document to a set of n-grams (n=5)
2MinHash: apply k hash functions to shingle set → k-dim signature vector
3LSH: bucket similar signatures together (Jaccard similarity estimation)
4Remove: keep one doc per near-duplicate cluster (Jaccard > 0.8)
Jaccard similarity
J(A,B) = |A ∩ B| / |A ∪ B|
MinHash estimates this in O(k) instead of O(|A|+|B|)

Phi-1 Key Finding (Gunasekar et al., 2023)
"Textbooks are all you need" — a 1.3B model trained on 1B synthetic textbook-quality tokens matched or exceeded 12B+ models trained on 100B web-scraped tokens. The synthetic data was generated by GPT-4 with explicit "write a clear, pedagogical explanation" prompts.
