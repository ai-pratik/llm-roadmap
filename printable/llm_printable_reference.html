Transformer Architecture
Attention Is All You Need · Vaswani et al., 2017 · arXiv:1706.03762
W1 · Core
Core Equations
Scaled Dot-Product Attention
Attention(Q,K,V) = softmax(QKᵀ / √dₖ) · V
Q = query, K = key, V = value, dₖ = key dimension. Divide by √dₖ to prevent vanishing gradients in softmax.
Multi-Head Attention
MultiHead(Q,K,V) = Concat(head₁,...,headₕ)Wᴼ
headᵢ = Attention(QWᵢQ, KWᵢK, VWᵢV)
Feed-Forward Block
FFN(x) = max(0, xW₁ + b₁)W₂ + b₂
Positional Encoding (Sinusoidal)
PE(pos,2i) = sin(pos / 10000^(2i/d))
PE(pos,2i+1) = cos(pos / 10000^(2i/d))
RoPE (Rotary PE — used in LLaMA/Mistral)
Apply rotation matrix R(θ·pos) to Q and K
f(x,m) = x · e^(imθ) [complex notation]

Key Interview Questions
Why divide by √dₖ in attention?
Without scaling, dot products grow large with dimension, pushing softmax into regions with near-zero gradients. √dₖ keeps variance stable.
Why multi-head instead of single-head attention?
Different heads learn different relationship types (syntactic, semantic, positional) simultaneously. Concatenating gives richer representations than one large attention.
What does the masking in decoder self-attention do?
Causal mask prevents token i from attending to tokens j>i — enforces autoregressive generation (can only use past context).

Scaling Laws + Modern Architectures
Kaplan 2020 · Chinchilla 2022 · MoE · GQA · Mamba SSM
W2–W3 · Core
Scaling Laws (Kaplan vs Chinchilla)
Kaplan 2020 — key finding
Loss ∝ N^(-0.076) · D^(-0.095) · C^(-0.050)
N=params, D=data tokens, C=compute
Kaplan: scale params faster than data. Led to GPT-3 (175B params, less data).
Chinchilla 2022 — revised finding
Optimal: N ∝ C^0.50, D ∝ C^0.50
Rule of thumb: ~20 tokens per parameter
Chinchilla: scale params AND data equally. Led to smaller but better-trained models (Llama-2, Mistral).
The disagreement: Kaplan used fixed data, Chinchilla optimized jointly. Chinchilla is now the standard — GPT-4 era models are all Chinchilla-optimal.

TypeKV headsMemoryQuality
MHA= Q headsHighBest

LLaMA-3 and Mistral use GQA with G=8 — best quality/memory tradeoff. KV cache scales with G, not H.
Mixture of Experts (MoE)
Routing function
G(x) = softmax(x · Wg) [router]
output = Σ G(x)ᵢ · Expertᵢ(x) [top-k only]
Mixtral uses top-2 routing from 8 experts. Active params per token = 2 experts. Total params = 8 experts but inference cost = 2 experts.
Top-kTypically k=2. Higher k = more compute, better quality
Load balanceAuxiliary loss prevents all tokens routing to same expert
Expert sizeEach expert = standard FFN block
Mixtral8 experts, top-2, 7B active / 47B total
Mamba / SSM Architecture
State Space Model recurrence
h'(t) = A·h(t) + B·x(t)
y(t) = C·h(t)
Mamba: A,B,C,Δ are input-dependent (selective)
Key difference from S4: Mamba makes B, C, and Δ functions of the input x — "selective" SSM. This is what enables it to filter irrelevant tokens.

Sheet 03 · Week 7–8
Fine-Tuning: LoRA · QLoRA · DoRA · DPO
Hu 2021 · Dettmers 2023 · Liu 2024 · Rafailov 2023
W7–W8 · Fine-Tuning
LoRA — Low-Rank Adaptation
Core idea
W' = W + ΔW = W + B·A
B ∈ ℝ^(d×r), A ∈ ℝ^(r×k), r ≪ min(d,k)
W is frozen. Only A and B are trained. At init: A ~ N(0,σ²), B = 0 (so ΔW=0 at start). Scale: α/r where α is a hyperparameter.
r (rank)4–64 typical. Higher = more capacity, more memory
alpha (α)Usually 2×r. Controls effective learning rate
targetq_proj, v_proj at minimum. All attention for best results
paramsAdds r×(d+k) params per layer. Much less than full fine-tune
QLoRA — 4-bit LoRA
Key components
NF4 quant.4-bit Normal Float — optimal for normally distributed weights
Double quantQuantize the quantization constants — saves extra memory
Paged optim.GPU→CPU memory offload for optimizer states on OOM spikes
Memory7B model: ~4GB (QLoRA) vs ~14GB (bf16) vs ~28GB (full)
DoRA — Weight-Decomposed LoRA
W = m · (V / ‖V‖) [magnitude × direction]
DoRA trains m (scalar) and ΔV (via LoRA)
Separating magnitude and direction stabilizes training. Empirically outperforms LoRA at same rank. Use: peft.LoraConfig(use_dora=True)
DPO — Direct Preference Optimization
RLHF objective (what DPO replaces)
max E[r(x,y)] - β·KL[π_θ ‖ π_ref]
DPO loss (closed-form solution)
L_DPO = -E[log σ(β·log(π_θ(y_w|x)/π_ref(y_w|x))
- β·log(π_θ(y_l|x)/π_ref(y_l|x)))]
y_w = chosen (winner), y_l = rejected (loser). No reward model needed — the policy IS the implicit reward model. β controls KL divergence penalty.
Key insight: the optimal RLHF policy can be expressed analytically in terms of log-probability ratios. DPO is exact RLHF, not an approximation.
SimPO — Reference-Free DPO
L_SimPO = -E[log σ(γ/|y_w| · log π_θ(y_w|x)
- γ/|y_l| · log π_θ(y_l|x) - γ)]
No reference model! Uses length-normalized log-probabilities + margin γ. Simpler, often better than DPO. β=2.0, γ=0.5 are good defaults.

RLHF Pipeline (InstructGPT)
1SFT: Fine-tune base model on high-quality human demonstrations (13k prompts in InstructGPT)
2Reward Model: Collect human comparisons (33k pairs). Train RM: base LLM + scalar head. Objective: r(y_w) > r(y_l)
3PPO: Optimize SFT model against RM using PPO. KL penalty to prevent reward hacking: r_total = r_RM - β·KL(π_θ ‖ π_SFT)
Reward Model Loss
L_RM = -E[log(σ(r_θ(x,y_w) - r_θ(x,y_l)))]
PPO Clipped Objective
L_CLIP = E[min(r_t·Â_t, clip(r_t, 1-ε, 1+ε)·Â_t)]
r_t = π_θ(a|s) / π_old(a|s)
InstructGPT found: RLHF model with 1.3B params was preferred over 175B SFT model. Alignment matters more than scale at inference.

1Red team: Model generates harmful responses to adversarial prompts

CAI key insight: you can generate preference data at scale without human labelers by using the model itself as an annotator, guided by explicit principles.

Sheet 07 · Week 21
FlashAttention + Distributed Training
FlashAttention-2 (Dao 2023) · Megatron-LM · ZeRO · DDP
W21 · Theory
Why Standard Attention is Slow
  • Standard attention materializes the full N×N attention matrix in HBM (GPU global memory)
  • HBM bandwidth is ~2TB/s — reading/writing N×N is the bottleneck, not FLOPs
  • For N=4096: attention matrix = 4096²×2 bytes = 32MB per head — bandwidth limited
FlashAttention — IO-Aware Tiling
1Tile Q, K, V into blocks that fit in SRAM (fast on-chip memory, ~20MB)
2Compute attention block by block — never write full N×N matrix to HBM
3Online softmax (log-sum-exp trick) — accumulate correct softmax across blocks
4Recomputation — during backward, recompute attention from Q,K,V instead of storing it
Memory complexity
Standard: O(N²) HBM reads/writes
Flash: O(N) HBM reads/writes ← key gain
FLOPs are the same. The speedup comes entirely from reducing HBM bandwidth usage. On H100: ~3× faster than standard attention at N=4096.
Parallelism Strategies
TypeWhat's splitBest for

Tensor (TP)Weight matrices across GPUsVery large layers

ZeRO — Memory Optimization Stages

ZeRO-3: each GPU holds only 1/N of parameters. Communication cost increases but enables training models that don't fit on any single GPU.
PyTorch DDP vs FSDP
DDPEach GPU holds full model. Gradients all-reduced. Simple. For models that fit on 1 GPU.
FSDPParameters sharded across GPUs (ZeRO-3 equivalent). For models > 1 GPU memory.
Grad accumAccumulate over N steps before optimizer update. Simulates N× larger batch.
Act. checkptDon't store activations for backward. Recompute them. ~1.4× slower, ~3× less memory.
Key Interview Questions
Explain why FlashAttention is faster without approximation.
It reduces HBM bandwidth usage via SRAM tiling. The bottleneck is memory bandwidth, not FLOPs. FlashAttention performs the same FLOPs but reads/writes O(N) instead of O(N²) to slow HBM.
What is activation checkpointing and when would you use it?
Instead of storing all forward activations for backward pass, recompute them on demand. Trades compute for memory. Use when GPU memory is the bottleneck — typically for large models or long sequences.
Sheet 08 · Week 22
Mechanistic Interpretability
Toy Models of Superposition · SAE · Activation Steering — Anthropic
W22 · Interpretability
Superposition Hypothesis
Neural networks represent more features than they have dimensions by encoding multiple features in superposition — overlapping, non-orthogonal directions. They "sacrifice" linear algebra cleanliness to store more information.
MonosemanticOne neuron → one concept (ideal, rarely found)
PolysemanticOne neuron → multiple unrelated concepts (common)
Superpositionn features stored in d dimensions where n >> d
InterferenceCost of superposition: features slightly activate each other
Superposition model (Toy Models paper)
x̂ = ReLU(W^T W x + b)
Loss = Σ importance(i) · (xᵢ - x̂ᵢ)²
W ∈ ℝ^(d×n), n > d. The model learns to pack n features into d dimensions. Rare, important features get the most "dedicated" directions.
Sparse Autoencoders (SAE)
1Hook into a layer's activation (residual stream, MLP out)
2Encoder maps to a larger sparse space: f(x) = ReLU(W_enc · x + b_enc)
3Decoder reconstructs: x̂ = W_dec · f(x) + b_dec
4Loss: reconstruction error + λ·L1(f(x)) [L1 enforces sparsity]
5Interpret each SAE feature by finding its top activating tokens
Activation Steering (ActAdd / RepE)
1Collect activations at layer L for concept A and concept B
2Steering vector: v = mean(act_A) - mean(act_B)
3During inference: h_L ← h_L + α·v
4Model behavior shifts toward concept A (or away from B)
Example: steering vector (French - English) added to residual stream causes model to generate French. Works for abstract concepts too: happiness, formality, refusal.
Key Interpretability Concepts
CircuitSubgraph of attention heads + MLPs implementing a specific behavior
Induction headTwo-head circuit that implements in-context learning via copying
Residual streamThe central "working memory" — each layer reads/writes to it
Logit lensProject residual stream at each layer to vocab — track prediction formation
ProbingTrain linear classifier on activations — test if concept is linearly encoded
Key Interview Questions
Why do LLMs need superposition? Why not just have more neurons?
Compute scales with d², but the number of features in the world scales with experience. Superposition is a compression strategy — the model stores more concepts than neurons by accepting small interference costs for rare features.
What is the goal of a sparse autoencoder in interpretability?
To decompose polysemantic neuron activations into monosemantic features. The SAE learns a larger but sparser basis where each direction (feature) corresponds to one interpretable concept.

10 Must-Know Explain-Out-Loud Answers
1. Why does attention use √dₖ scaling?
Dot products grow with dimension, pushing softmax into low-gradient saturation regions. √dₖ keeps variance ~1 regardless of d.
2. Why is DPO equivalent to RLHF?
The optimal RLHF policy has a closed-form solution expressible in log-probability ratios. DPO directly optimizes this form, bypassing the reward model entirely.
3. How does FlashAttention achieve O(N) memory?
By tiling Q, K, V into SRAM blocks and computing attention block-by-block, it never materializes the full N×N matrix in slow HBM memory.
4. What problem does LoRA solve?
Full fine-tuning requires gradients for all parameters (~14GB for 7B model). LoRA freezes the base model and adds trainable low-rank matrices (~50MB), reducing memory 99%.
5. What is superposition in neural networks?
Storing more features than neurons by representing features as non-orthogonal directions. Trade-off: more capacity, but features interfere with each other (polysemanticity).
6. What is the difference between Kaplan and Chinchilla?
Kaplan (2020) found scale parameters faster than data. Chinchilla (2022) showed this was wrong — optimal is to scale both equally at ~20 tokens/param. GPT-3 was undertrained.
7. Why does Constitutional AI not need human labelers?
The model critiques its own outputs against written principles, generating preference data automatically (RLAIF). This scales to millions of examples at near-zero cost.
8. What is PagedAttention and why is it faster?
Manages KV cache like OS virtual memory — non-contiguous blocks, no fragmentation. With continuous batching, GPU utilization approaches 100% vs ~40% for naive serving.
9. What is the difference between a process reward model and outcome reward model?
ORM scores the final answer only. PRM scores each reasoning step — catches "right answer, wrong reasoning" which ORMs miss. Critical for math and code tasks.
10. What makes an MoE model efficient at inference?
Only top-k experts activate per token. Mixtral has 47B total params but only 12B active per forward pass — you get large model capacity at medium model compute cost.
